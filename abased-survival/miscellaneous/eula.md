# EULA Compliance

All our server setups and configurations are EULA-compliant right from the start. However, once you make changes to the setup, we cannot guarantee continued compliance. It is important to review and maintain EULA compliance after any modifications.

You should read this page to learn some important information about keeping your server EULA Compliant.

### For more information, please refer to the links below: <a href="#useful-links" id="useful-links"></a>

* **Minecraft EULA:** [https://www.minecraft.net/en-us/eula](https://www.minecraft.net/en-us/eula)
* **Minecraft Usage Guidelines:** [https://www.minecraft.net/en-us/usage-guidelines](https://www.minecraft.net/en-us/usage-guidelines)

### Crate Reward Chances <a href="#crate-reward-chances" id="crate-reward-chances"></a>

In setups that include crates, the chances of winning specific items are displayed in the crate previews. This is required by the Minecraft EULA and Commercial Usage Guidelines. Be sure to keep these chances visible in your server to maintain compliance.

### Server Disclaimer: Mojang and Microsoft <a href="#microsoft-association" id="microsoft-association"></a>

You must clearly state on your server's website or store that your server is not affiliated with or approved by Mojang or Microsoft.

### Selling Tokens in your store <a href="#selling-coins-in-your-store" id="selling-coins-in-your-store"></a>

Many of our pre-made server setups feature Tokens, a premium currency you can sell for real money. According to the Minecraft [Usage Guidelines](https://www.minecraft.net/en-us/usage-guidelines), you must provide users with all pricing and content information related to your server before they join or sign in.

When selling Tokens in your store, you must explain how they can be used, not just in in-game menus but also on your website. We’ve provided templates you can use for this purpose—be sure to include these details in the description of your "Tokens" (or equivalent) package.

Since each setup offers different items, choose the correct template based on the server setup you're using.
